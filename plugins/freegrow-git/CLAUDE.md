# Freegrow Git 워크플로우 규칙

이 프로젝트는 **freegrow-git 플러그인**을 사용합니다. 아래 규칙을 **반드시** 따르세요.

## 커밋 규칙 (필수)

"커밋해", "커밋", "commit" 요청 시 **반드시** 아래 형식을 따릅니다:

```
<type>: <제목 요약> (#이슈번호)

- <변경 내용 1>
- <변경 내용 2>
- 기존: ... → 변경: ... (해당 시)
- 변경 파일:
  - <파일1> - <변경 설명>
  - <파일2> - <변경 설명>

💯 Generated by freegrow-git plugin
```

### 커밋 작성 절차

1. `git diff --staged` 분석하여 변경 내용 파악
2. 브랜치에서 이슈 번호 추출 (feature/3-xxx → #3)
3. 상세 본문 작성 (제목만 있는 커밋 금지)
4. 변경 파일 목록 포함
5. 시그니처 추가: `💯 Generated by freegrow-git plugin`

### 타입 종류

| 타입 | 설명 |
|------|------|
| `feat` | 새로운 기능 |
| `fix` | 버그 수정 |
| `docs` | 문서 변경 |
| `style` | 코드 포맷팅 |
| `refactor` | 리팩토링 |
| `test` | 테스트 |
| `chore` | 빌드, 설정 |

## PR/이슈 규칙

- **Assignee**: `@me` (현재 로그인된 사용자)
- **기본 브랜치**: `develop`
- **PR 로그 파일 포함**: `.claude/pr-logger/session-log-*.md`

## 커맨드 (선택)

| 커맨드 | 설명 |
|--------|------|
| `/freegrow-git:start` | 이슈 + 브랜치 생성 |
| `/freegrow-git:com` | 커밋 (이 규칙과 동일) |
| `/freegrow-git:pr` | PR 생성 |
