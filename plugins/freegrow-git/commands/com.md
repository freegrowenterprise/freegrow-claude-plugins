# Freegrow 커밋

Freegrow 팀 컨벤션에 맞는 **상세한 커밋 메시지**를 작업 단위별로 생성합니다.

## 인자

`$ARGUMENTS` - 커밋 제목 또는 빈 값 (자동 분석)

## 실행 절차

### 1. 현재 상태 확인

```bash
git status
git diff --stat
git diff --staged --stat
```

스테이징된 파일이 없고 수정된 파일도 없으면 "커밋할 내용 없음"으로 종료합니다.

### 2. 이슈 번호 확인

현재 브랜치에서 이슈 번호를 추출합니다:
```bash
git branch --show-current
```

브랜치 형식이 `feature/123-description`이면 이슈 번호는 `123`입니다.
이슈 번호를 찾을 수 없으면 사용자에게 직접 입력받습니다.

### 3. 작업 단위 분석 (핵심)

**모든 변경 파일을 분석하여 논리적 작업 단위로 그룹화합니다.**

```bash
git diff --stat
git diff
```

**작업 단위 분류 기준:**

1. **기능 단위**: 하나의 기능과 관련된 파일들을 묶음
   - 예: 로그인 기능 → `login_screen.dart`, `auth_service.dart`, `auth_provider.dart`
2. **설정/인프라 변경**: 빌드, CI/CD, 설정 파일
   - 예: `.github/workflows/`, `pubspec.yaml`, `Makefile`
3. **문서 변경**: README, 문서 파일
   - 예: `README.md`, `CHANGELOG.md`, `docs/`
4. **리팩토링**: 기능 변경 없는 코드 개선
**작업 단위 분류 방법:**
- 같은 디렉토리/모듈의 관련 파일은 하나의 단위
- 서로 다른 목적의 변경은 별도 단위로 분리
- 변경 내용(diff)을 분석하여 논리적 연관성 판단

### 4. 작업 단위별 커밋 계획 수립

분석 결과를 바탕으로 커밋 계획을 세웁니다:

```
📋 커밋 계획:
1. [chore] GitHub Actions 자동 릴리스 워크플로우 추가
   - .github/workflows/auto-release.yml
2. [chore] 레포지토리 이름 변경 반영
   - .claude-plugin/marketplace.json
   - README.md
   - commands/plugin-install.md
   - commands/plugin-setup.md
```

**사용자에게 커밋 계획을 보여주고 확인받습니다.**

변경 파일이 모두 하나의 작업 단위에 속하면 단일 커밋으로 진행합니다.

### 5. 작업 단위별 커밋 실행

각 작업 단위에 대해 순서대로:

#### 5-1. 해당 파일만 스테이징

```bash
git add <파일1> <파일2> ...
```

#### 5-2. 변경 내용 분석

```bash
git diff --staged
```

#### 5-3. 커밋 타입 결정

| 타입 | 사용 시점 |
|------|----------|
| `feat` | 새 기능 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서 변경 (.md 파일) |
| `style` | 포맷팅, 세미콜론 등 |
| `refactor` | 기능 변경 없는 코드 개선 |
| `test` | 테스트 추가/수정 |
| `chore` | 빌드, 설정 파일 변경 |

#### 5-4. 상세 커밋 메시지 생성

**커밋 메시지는 반드시 다음 형식을 따릅니다:**

```
<type>: <제목 요약> (#이슈번호)

- <변경 내용 1>
- <변경 내용 2>
- <변경 내용 3>
- 변경 파일:
  - <파일1> - <변경 설명>
  - <파일2> - <변경 설명>

💯 Generated by freegrow-git plugin
```

**상세 메시지 작성 규칙:**
1. **제목**: 핵심 변경 사항을 한 줄로 요약 (50자 이내)
2. **본문**:
   - 무엇이 변경되었는지 불릿 포인트로 설명
   - 기존 동작 vs 변경된 동작 설명 (해당 시)
   - 주요 로직 변경 설명
3. **변경 파일**: 각 파일별 변경 내용 간략 설명

#### 5-5. 커밋 실행

```bash
git commit -m "$(cat <<'EOF'
<type>: <제목 요약> (#이슈번호)

- 변경 내용 설명
- 변경 파일:
  - file1 - 설명

💯 Generated by freegrow-git plugin
EOF
)"
```

### 6. 전체 결과 출력

모든 커밋 완료 후 전체 결과를 출력합니다:

```bash
git log --oneline -<커밋 수>
```

```
✅ 커밋 완료! (총 N개)

1. abc1234 chore: GitHub Actions 자동 릴리스 추가 (#2)
2. def5678 chore: 레포지토리 이름 변경 반영 (#2)
```

## 예시: 작업 단위 분리

### 입력 상태
```
수정됨: .github/workflows/auto-release.yml (신규)
수정됨: .claude-plugin/marketplace.json
수정됨: README.md
수정됨: commands/plugin-install.md
수정됨: commands/plugin-setup.md
```

### 커밋 결과
```
커밋 1: chore: GitHub Actions 자동 릴리스 워크플로우 추가 (#2)
커밋 2: chore: 레포지토리 이름 변경 반영 (#2)
```

## 예시: 단일 작업

모든 변경이 하나의 작업에 속하면 기존처럼 단일 커밋:

```
feat: 바텀시트 실시간 UI 업데이트 구현 (#10)

- 앵커/태그 상세 바텀시트에 Consumer 적용
- 기존: 바텀시트 오픈 시 전달받은 데이터로만 표시 (정적)
- 변경: provider 상태 watch하여 실시간 데이터 반영 (동적)
- 변경 파일:
  - device_list_screen.dart - Consumer로 래핑, 실시간 업데이트
  - coordinate_screen.dart - 태그 바텀시트 동적 렌더링

💯 Generated by freegrow-git plugin
```

## 사용 예시

```
/freegrow-git:com                    # 변경 내용 분석 → 작업 단위별 커밋
/freegrow-git:com feat: 로그인 구현   # 단일 커밋 (제목 지정 시)
```

## 주의사항

- 이슈 번호 없이는 커밋 불가
- **반드시 상세한 본문 포함** (제목만 있는 커밋 금지)
- 변경 파일 목록 필수 포함
- 커밋 계획을 사용자에게 먼저 보여주고 확인받기
