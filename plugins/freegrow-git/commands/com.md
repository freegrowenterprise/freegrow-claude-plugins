# Freegrow 커밋

Freegrow 팀 컨벤션에 맞는 **상세한 커밋 메시지**를 생성합니다.

## 인자

`$ARGUMENTS` - 커밋 제목 또는 빈 값 (자동 분석)

## 실행 절차

### 1. 현재 상태 확인

```bash
git status
git diff --staged --stat
git diff --staged
```

스테이징된 파일이 없으면 `git add .` 실행 여부를 사용자에게 확인합니다.

### 2. 이슈 번호 확인

현재 브랜치에서 이슈 번호를 추출합니다:
```bash
git branch --show-current
```

브랜치 형식이 `feature/123-description`이면 이슈 번호는 `123`입니다.
이슈 번호를 찾을 수 없으면 사용자에게 직접 입력받습니다.

### 3. 변경 내용 분석

**반드시 `git diff --staged`를 분석하여 다음을 파악합니다:**
- 어떤 기능이 추가/수정/삭제되었는지
- 주요 변경 로직
- 변경된 파일과 각 파일의 변경 내용

### 4. 커밋 타입 결정

변경 내용을 분석하여 적절한 타입을 결정합니다:

| 타입 | 사용 시점 |
|------|----------|
| `feat` | 새 기능 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서 변경 (.md 파일) |
| `style` | 포맷팅, 세미콜론 등 |
| `refactor` | 기능 변경 없는 코드 개선 |
| `test` | 테스트 추가/수정 |
| `chore` | 빌드, 설정 파일 변경 |

### 5. 상세 커밋 메시지 생성

**커밋 메시지는 반드시 다음 형식을 따릅니다:**

```
<type>: <제목 요약> (#이슈번호)

- <변경 내용 1>
- <변경 내용 2>
- <변경 내용 3>
- 변경 파일:
  - <파일1.dart> - <변경 설명>
  - <파일2.dart> - <변경 설명>

💯 Generated by freegrow-git plugin
```

**상세 메시지 작성 규칙:**
1. **제목**: 핵심 변경 사항을 한 줄로 요약 (50자 이내)
2. **본문**:
   - 무엇이 변경되었는지 불릿 포인트로 설명
   - 기존 동작 vs 변경된 동작 설명 (해당 시)
   - 주요 로직 변경 설명
3. **변경 파일**: 각 파일별 변경 내용 간략 설명

### 6. PR 로그 파일 포함

`.claude/pr-logger/session-log-*.md` 파일이 수정되었으면 함께 스테이징합니다:
```bash
git add .claude/pr-logger/
```

### 7. 커밋 실행

**HEREDOC을 사용하여 멀티라인 커밋 메시지를 작성합니다:**

```bash
git commit -m "$(cat <<'EOF'
<type>: <제목 요약> (#이슈번호)

- 변경 내용 설명
- 추가 설명
- 변경 파일:
  - file1.dart - 설명
  - file2.dart - 설명

💯 Generated by freegrow-git plugin
EOF
)"
```

### 8. 결과 출력

커밋 전체 내용을 출력합니다:
```bash
git log -1
```

## 예시: 상세 커밋 메시지

```
feat: 바텀시트 실시간 UI 업데이트 구현 (#10)

- 앵커/태그 상세 바텀시트에 Consumer 적용
- 기존: 바텀시트 오픈 시 전달받은 데이터로만 표시 (정적)
- 변경: provider 상태 watch하여 실시간 데이터 반영 (동적)
- 변경 파일:
  - device_list_screen.dart - Consumer로 래핑, 실시간 업데이트
  - coordinate_screen.dart - 태그 바텀시트 동적 렌더링

💯 Generated by freegrow-git plugin
```

## 사용 예시

```
/freegrow-git:com                    # 변경 내용 분석 후 자동 생성
/freegrow-git:com feat: 로그인 구현   # 제목 제공, 본문은 자동 생성
```

## 주의사항

- 이슈 번호 없이는 커밋 불가
- PR 로그 파일 자동 포함
- **반드시 상세한 본문 포함** (제목만 있는 커밋 금지)
- 변경 파일 목록 필수 포함
