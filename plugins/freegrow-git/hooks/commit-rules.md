# Freegrow 커밋 규칙 (조건부 적용)

## 문맥 판단 (중요)

먼저 사용자의 요청이 **커밋 생성** 요청인지 판단하세요:

**커밋 생성 요청 (이 규칙 적용):**
- "커밋해", "커밋하고 푸시해", "변경사항 커밋해줘"

**커밋 생성이 아닌 요청 (이 규칙 무시):**
- "커밋 기록 확인해", "커밋 히스토리 보여줘", "커밋 로그 봐줘"
- "커밋 메시지 수정해", "커밋 취소해", "커밋 되돌려"

**커밋 생성 요청이 아니면 아래 규칙을 무시하고 요청에 맞게 처리하세요.**

---

## 커밋 생성 시 필수 형식

```
<type>: <제목 요약> (#이슈번호)

- <변경 내용 1>
- <변경 내용 2>
- 기존: ... → 변경: ... (해당 시)
- 변경 파일:
  - <파일1> - <변경 설명>
  - <파일2> - <변경 설명>

🔍 리뷰 포인트:
- <리뷰어가 주의 깊게 봐야 할 사항>

💯 Generated by freegrow-git plugin
```

## 커밋 작성 절차

1. `git diff --staged` 분석하여 변경 내용 파악
2. 브랜치에서 이슈 번호 추출 (`feature/3-xxx` → `#3`)
3. 상세 본문 작성 (**제목만 있는 커밋 금지**)
4. 변경 파일 목록 포함
5. 리뷰 포인트 작성 (**필수** - 리뷰어가 놓칠 수 있는 사항)
6. 시그니처 추가: `💯 Generated by freegrow-git plugin`
7. PR 로그 파일 포함: `.claude/pr-logger/session-log-*.md`

## 타입

| 타입 | 설명 |
|------|------|
| `feat` | 새로운 기능 |
| `fix` | 버그 수정 |
| `docs` | 문서 변경 |
| `refactor` | 리팩토링 |
| `chore` | 빌드, 설정 |
